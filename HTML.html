HTML - Getting started

On 26 Jan 2017

Out of the three important web design languages, HTML has the utmost importance. CSS and JavaScript may control how a webpage looks, but it is HTML that decides on the page's structural contents. This article will guide you through some simple instructions to start a webpage.

Basic page structure

HTML tags require an opening tag and a closing tag to work.

The <head> tag includes a page's basic information and appear at the top of an HTML document. This area is invisible when viewed in the browser.

The DOCTYPE declaration lets the user agents know which version of HTML is being used.

The <meta> tag describes the UTF encoding that will be used.

The <title> tag declares the page's title. Every page on a website should have a unique title.

The <body> tag indicates a page's body. The body contains everything that you see in a webpage when opened on a browser.

Formatting Page Content

The heading tags, <h1> to <h6>, serve the purpose of arranging the webpage into a hierarchy structure. The size of each heading gets smaller as the tag number goes up.

Aside from headers, paragraph (<p>) may be the most basic formatting tag. If you don't put a text inside a tag, your browser will automatically assume it to be a paragraph. Since a paragraph may get very long and you may forget to close the <p> tag, the advice here is to open up and close a tag before putting content in between.

Some tags don't need to be closed, and <br> is one of those. Line break tag is used when you want to forcefully skip down a line.

Emphasizing tags include, but are not limited to, <b>, <strong>, <i> and <em>. The first two will make text goes bold, while the latter two make text becomes italic. The difference is that <strong> and <em> will be recognized as an emphasis by screen reader or other accessibility tools.

Named character entity is used for displaying special characters that are not on the usual keyboard, or may otherwise cause errors in an HTML file. A prime example of such error would be the angle brackets "<" and ">". To display those characters properly without any issues, you need to start with the ampersand (&), then the name entity for that particular character, and closing with a semicolon (;). Example of how to type & in an HTML document: &amp;.

Non-breaking space, or &nbsp;, is used when you need to add more space into your document without spamming spacebar. It will create white spaces but will not break down to a new line.

To display images, the tag <img> is used. The most basic syntax should be <img src="#" >, with the image's direct link replacing the #. The image tag doesn't need to have a closing tag. Usually the parameter title=" " is used to let screen reader knows what kind of image it is.

Structuring Content

The structural content of a page created by HTML allows us to understand the page's meaning better. This semantics may not look any different to us, but for those who do not rely on visual use, search engines for example, it is very important to classify webpage information in a logical manner.

<nav> elements are used to indicate navigation within your websites. They are usually found at menus.

<article> represents a complete self-contained composition in a document, page, application or site. <section> represents a generic section of a document. These two tags often cause confusion for web designers, so it all depends on your own judgement as to where to use them.

<aside> is a new element in HTML5 that is equivalent to the content within a sidebar. It can also be used for typographic effects like pull quotes, or advertisements.

<div> is used when you don't really need any semantic meaning for your content. It is a very popular tag, and is intended for when you want to group contents together for stylistic purpose. It can be used anywhere within the document.

<header> typically contains a group of introductory or navigational aids. For introduction purpose, it represents information on its nearest parent sectioning content. If it doesn't have a parent element then it represents the whole page.

Creating Links

<a href="#"> is the simplest form of an anchor tag - a tag with link leading to any other places on the web or within the website, or even webpage. The anchor tag can have many other attributes. target=" " attribute controls how a new page is opened: in a new window or new tab. The rel=" " attribute defines the relationship between the target object and the link object. The title=" " attribute gives the link a title, which makes it more accessible to assistive devices or search engines.

To link to pages within the website, instead of a full web address, you only need to indicate the path leading to where the page or file is located, starting from where your current page is. Using "folder_2/page2.html" means your current page (page1.html) in the same directory as folder_2, and the page page2.html is within that folder. Using the same example, when you use the link "../page1.html" on page page2.html, it will lead you back to page1.html.

For external links, the one important thing that you should always remember is to include the protocol (usually http or https) with the link, or else it won't work.

On the occasion that you need to link to a downloadable resource, you can give your anchor tag the attribute download. The attribute will force your browser to download the file, even if it can be read by the browser.

In order to jump to a specific point within a page, first you need to assign a unique id for the destination point, then use the normal anchor tag with #id instead of the normal link address.
