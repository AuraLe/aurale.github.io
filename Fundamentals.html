Web Technologies Fundamentals

On 19 Jan 2017

Understanding how the web works and which technologies power it will explain the course of action that a web designer may take. Even if you don't plan on becoming a web designer, learning web terminologies will benefit you in the long run due to the popularity of Internet.

Introduction to the web

The Internet is different from the Web. Internet is a massive network of computers around the world while the Web is way to transmit data over the internet.

Transmiting data refers to the interaction between clients and servers (clients request data, server deliver data).

When you access an URL, the browser sends request to DNS. DNS translates URL into IP address. Then the browser uses IP address to contact server to ask for contents. Content is eventually returned and displayed on browser.

IP address is easy for computers to work with, but human can’t remember all of them. Therefore, domain name was created as the human’s version of a site’s unique ID, or IP address. DNS is used to translate IP address from and to Domain name (Internet phone book).

Protocols are simple languages that any computer can use to communicate with each other, regardless of their spec. Different layers of protocol are needed for different tasks.

HTTP acts as the standard protocol to transmit resources on the web. Once a request is sent and received, it’s discarded and forgotten. With such simple and efficient protocol, web has no burden to remember.

URL consists of multiple parts. For example, for the link http://www.lynda.com, http is the protocol, www is subdomain, lynda is domain name, com is top level domain.

Subdomain refers to the default “folder” that contains most web contents on web server. Proper URL format is important for the site to work.

Browsers differ by rendering engines. Web designers need to understand how browsers work in order to ensure that websites work the properly on each browser. In fact, designers should test their pages on multiple browsers and multiple versions of the same browser.

World Wide Web Consortium - W3C: standard web protocols in technologies to ensure the consistency of web designs across different browsers and devices.

Web server is where website is hosted, usually a computer with specialized set of software and components. Each server has its own stack. Stack means operating system + web server + database server + programming language.

Front-end Technology

Front-End is the UI layer of website, created with HTML, CSS, and JavaScript. A front-end designer works more like a visual designer than a programmer.

HTML - HyperText Markup Language is the standard markup language for web pages. It is simple and essential for any web designer. Latest standard is HTML5, which is compatible with older versions while providing new, advanced features. HTML is used to define the structure of the page.

CSS - Cascading Style Sheet means the style sheet language used to control the presentation of HTML. It defines the format and style of the page.

JavaScript: the scripting language that adds interactivity and application functionality to sites (dynamic menus, open new window, update data in the browser, etc.). JavaScript is client-side and doesn’t need to be refreshed to work.

JavaScript framework: prewritten JavaScript functions to simplify the designing process for developers.

Common image types include JPEG, GIF, PNG and SVG.

JPEG
Widely used. Lossy compression to reduce size. Lower size means lower quality.
GIF
Most common on the web. Limited to 256 colors. Lossless, but limited color tones. Usually used in solid or flat color artworks. Support transparency and animation, so it's used widely for icons and animations.
PNG
Support transparency. Use lossless compression format. Support more colors like JPEG. Have larger size comparing to JPEG and GIF.
SVG
Not popular (yet). Vector format > scale to any size without losing quality. Can be edited with web code and CSS properties. Larger in size comparing to other types.

API, or "Application Programming Interfaces", provides a set of instructions and standards for using a specific service. API is constantly changing and needs to be updated constantly. API from smaller, lesser known sites can be unstable. HTML5 has allowed higher-level APIs to do specific tasks.

HTML5 intergrates web applications into HTML (video, animations, plugins, etc.). APIs are intergrated into HTML5 to keep HTML5 from becoming too bulky.
